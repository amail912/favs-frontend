# Guideline de documentation
Ce document sert à donner des instructions pour un agent IA pour la rédaction de documentation.

## Documentation layout
- La documentation se trouve dans le dossier `docs`.
- Le fichier `docs/features/features.md` liste les features du logiciel et indique l'emplacement du fichier qui décrit la feature.
- Le sous-dossier `docs/guidelines` définit un ensemble de documents pour accomplir certaines tâches comme le présent document.
- Le sous-dossier `docs/features` documente les différentes features présentent dans le logiciel.

## Instructions
- La documentation est en français. On veillera à bien utiliser les caractères diacritiques et les apostrophes là où ils sont nécessaires.
- À chaque changement de code, tu décideras si un changement de documentation est nécessaire. Tu informeras l'utilisateur de ta décision.
- La documentation doit rester synchronisée avec le code. En d'autres termes, ce qui est inscrit dans la documentation doit se retrouver dans le code, ce qui est dans le code doit être documenté.
