# Month View Presence Cues

## Goal
As a calendar owner, I want shared presence to extend to Month view so I can notice larger travel patterns over time.

## Why This Is Postponed
Month view has much less space than Day view and will likely need a more summarized presence language. The current iteration should first validate the trip model, frontend derivation, and Day view cue before deciding how much location detail Month view can carry.

## Re-entry Notes
- Reuse the same derived presence source as Day view.
- Decide whether Month view should expose current place, dominant place, or change markers only.
- Confirm that inspection remains usable on mobile and keyboard without overcrowding the grid.
